# UI 链路探究到稳定脚本

## 契约

```yaml
目标: 可验证的_UI_自动化脚本
输入优先级:
  - 当前界面证据
  - 用户提示或纠正
  - 官方或本地知识库
  - 既有脚本
  - 模型推断
不变量:
  - 没有干净运行证据不得声明稳定
  - 一次成功不得扩大支持范围
  - 不覆盖用户无关脚本或项目
  - 未验证能力必须显式拒绝
```

## 流程

| 步骤 | 动作 | 判据 | 失败处理 |
| --- | --- | --- | --- |
| 1 | 定义目标界面/项目状态 | 目标状态可观察、可测试 | 停止并先定义成功判据 |
| 2 | 收集知识库、用户提示、当前界面事实 | 每条结论有来源：`kb/user/ui/script` | 无来源结论标为假设 |
| 3 | 拆成最小状态转换 | 每段有前置、动作、观察 | 不写连续大脚本 |
| 4 | 在一次性项目/窗口中探测一个转换 | 有截图、UIA、日志、JSON 或文本证据 | 先分类失败再重试 |
| 5 | 只脚本化已验证转换 | 脚本输出结构化结果和证据路径 | 仅标为探针脚本，不标稳定 |
| 6 | 组合已验证转换 | 干净运行到达目标状态 | 删除或拒绝未验证分支 |
| 7 | 用独立判据验证 | 编译、导出、回读、保存重开或结果文本通过 | 不声明稳定 |
| 8 | 记录接口和边界 | 命令、参数、证据、错误、不支持项明确 | 不发布为稳定能力 |

## 状态记录

使用任务本地记录。保持机器可读。

```yaml
界面链路:
  任务目录: <一次性路径>
  目标状态: <可观察终态>
  当前状态: <窗口/项目/模式>
  状态转换:
    - id: T01
      前置: <状态>
      动作: <键盘/UIA/Win32/鼠标/脚本>
      观察: <期望界面或产物>
      证据: <路径>
      状态: 假设|已验证|失败|已拒绝
  稳定脚本:
    - 路径: <脚本>
      支持输入: [...]
      产物: [...]
      判据: <通过条件>
  禁止入口:
    - 入口: <错误入口或动作>
      原因: <具体失败>
      替代: <已验证链路或停止>
  边界:
    - 能力: <不支持项>
      行为: 拒绝|仅探针|需要用户
```

## 用户提示处理

```pseudo
for 提示 in 用户提示:
  记录(提示)
  假设 = 拆成界面状态转换(提示)
  if 假设 与 当前界面证据 冲突:
    优先(用户纠正, 当前界面证据)
  验证一个转换(假设)
  if 已验证:
    转成脚本步骤(假设)
  else:
    记录为失败或阻塞(假设)
```

规则：

| 情况 | 动作 |
| --- | --- |
| 用户纠正入口或动作 | 立即停止使用旧入口 |
| 用户指出当前模式不能编辑 | 编辑前先检测、恢复或重启 |
| 用户给出局部链路 | 先验证该片段，用它构造下一步探针环境 |
| 用户要求“先成功再脚本” | 先跑手工/探针路径；只脚本化已验证片段 |

## 脚本晋级

```yaml
晋级:
  探针脚本:
    允许条件:
      - 一个状态转换已验证
      - 已输出证据路径
    禁止声明:
      - 稳定
      - 通用设备支持
  稳定脚本:
    必需条件:
      - 干净一次性项目运行通过
      - 无手工补救
      - 输出结构化结果 JSON
      - 独立判据通过
      - 未支持输入会被拒绝
      - 不覆盖无关文件
```

结果 JSON 形状：

```json
{
  "ok": true,
  "chain": "verified-ui-chain-id",
  "inputs": {},
  "selected_ui": {},
  "artifacts": [],
  "oracle": {
    "name": "compile/export/readback/save_reopen",
    "passed": true
  },
  "unsupported": []
}
```

## 证据判据

| 声明 | 可接受判据 |
| --- | --- |
| 窗口已打开 | 标题 + UIA/Win32 身份 |
| 设备/列表项已选中 | 动作前捕获选中文本或 ID |
| 字段已配置 | 值回读或已保存配置产物 |
| 项目已改变 | 保存项目 + 重开/导出/回读 |
| 程序/配置可编译 | 官方编译/转换结果文本 |
| 脚本稳定 | 干净运行结果 JSON + 独立判据 |

截图只能辅助；不能单独证明稳定。

## 禁止项

| 模式 | 替代 |
| --- | --- |
| 把相似菜单当成同一操作 | 验证目标对象确实改变 |
| 基于未验证界面猜测写完整脚本 | 先探测一个状态转换 |
| 无新证据就增加 sleep | 检查状态、焦点、窗口身份 |
| 从单一设备扩大为通用支持 | 写明已验证型号和路径 |
| 静默尝试未验证导入/注册 | 用稳定错误码拒绝 |
| 用当前脏界面状态证明成功 | 在一次性干净项目重跑 |
| 覆盖用户已有脚本 | 新增脚本或只做局部补丁 |

## 示例

正确入口：

```yaml
提示: "项目树 -> 单元配置 -> CPU -> EtherNet/IP"
错误入口: "通信设置"
规则:
  动作: 使用项目树单元配置
  拒绝: 工具栏或菜单中的通信设置
判据:
  - 目标窗口标题包含 EtherNet/IP 设定
  - 配置设备出现在项目或单元状态中
```

先片段后链路：

```yaml
已验证片段:
  动作: 选中设备叶子项后按 Enter
  证据: 捕获 selected_leaf + 设定窗口更新
下一步:
  - 为该片段生成脚本
  - 重跑片段，给下一个弹窗突破创建稳定状态
禁止:
  - 等完整流程全部理解后才开始脚本化
```

边界：

```yaml
能力: 自动文件注册
状态: 未稳定
脚本行为:
  if input.esi_path:
    throw: UI_CHAIN_UNSTABLE_CAPABILITY
允许声明: "仅支持已注册设备"
禁止声明: "支持自动注册"
```

## 最终交付

```yaml
交付:
  脚本:
    - 路径
    - 命令
    - 参数
    - 不支持输入
  证据:
    - 干净运行结果
    - 判据产物
  文档:
    - 已验证链路
    - 禁止入口
    - 边界
  git:
    - 若修改了脚本或技能则提交
```
